# 장애등급 분류기준

## 개요

IT 서비스 장애 발생 시 **신속한 판단과 대응**을 위해 장애등급을 4단계로 분류합니다. 등급에 따라 대응 체계, 보고 라인, SLA 기준이 달라집니다.

## 등급 정의

| 등급 | 명칭 | 기준 | SLA (복구목표) | 보고 대상 |
|------|------|------|----------------|-----------|
| **P1** | 긴급 | 전사 서비스 중단, 매출 직접 영향 | 1시간 이내 | CTO + 경영진 |
| **P2** | 중대 | 핵심 서비스 일부 장애, 다수 사용자 영향 | 4시간 이내 | 본부장 + 팀장 |
| **P3** | 보통 | 단일 서비스 성능 저하, 우회 가능 | 1영업일 이내 | 팀장 |
| **P4** | 경미 | UI 오류, 비핵심 기능 불편 | 3영업일 이내 | 담당자 |

## P1 장애 판정 기준 (상세)

다음 중 **하나라도** 해당되면 P1으로 분류합니다:

1. 주문/결제 시스템 전면 중단
2. 전사 네트워크 또는 VPN 장애
3. DB 마스터 장애로 쓰기 불가
4. 보안 침해 사고 (데이터 유출 의심)
5. 동시 영향 사용자 1,000명 이상

## 에스컬레이션 절차

```
발견 → 초기 판단(5분) → 등급 결정 → 대응팀 소집
  ├─ P1: 즉시 전화 + 슬랙 #incident-war-room
  ├─ P2: 15분 내 슬랙 + 이메일
  ├─ P3: 1시간 내 티켓 생성
  └─ P4: 당일 내 티켓 생성
```
