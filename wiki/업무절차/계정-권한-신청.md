# 계정 및 권한 신청

## 신청 채널

- **인트라넷**: IT지원 > 계정/권한 신청
- **긴급**: 슬랙 #it-helpdesk (업무시간 외 당직자 대응)

## 시스템별 권한 체계

### 서버/클라우드

| 권한 레벨 | 설명 | 승인자 |
|-----------|------|--------|
| Viewer | 읽기 전용 (로그, 메트릭) | 팀장 |
| Operator | 서비스 재시작, 배포 | 팀장 + 인프라팀 |
| Admin | 리소스 생성/삭제, 설정 변경 | 본부장 + 보안팀 |

### 데이터베이스

| 권한 레벨 | 설명 | 승인자 |
|-----------|------|--------|
| Read Only | SELECT 권한 | 팀장 |
| Read/Write | INSERT, UPDATE, DELETE | 팀장 + DBA |
| DDL | 스키마 변경 | DBA + 보안팀 |
| DBA Admin | 전체 관리 권한 | CTO |

### GitHub/GitLab

| 역할 | 설명 | 승인자 |
|------|------|--------|
| Read | 코드 읽기 | 자동 (입사 시) |
| Write | Push, PR 생성 | 팀장 |
| Maintain | Branch protection 관리 | 테크리드 |
| Admin | Repo 설정 변경 | 개발본부장 |

## 권한 신청 시 필수 정보

1. **대상 시스템**: 어떤 시스템/서버/DB에 접근하려는지
2. **필요 권한 레벨**: 위 표 참고
3. **사유**: 구체적 업무 목적
4. **기간**: 상시 / 임시 (종료일 명시)
5. **IP 제한**: 특정 IP에서만 접근 필요 여부

## 권한 회수

- **퇴사자**: 인사팀 통보 즉시 전체 권한 회수 (당일)
- **부서 이동**: 이전 부서 권한 7일 내 회수
- **임시 권한**: 지정 기간 종료 시 자동 회수
- **분기 리뷰**: 매 분기 미사용 권한 일괄 정리
