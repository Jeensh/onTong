# 신규 서버 요청 절차

## 절차 요약

```
요청서 작성 → 팀장 승인 → 인프라팀 검토 → 리소스 할당 → 구성 완료 → 인수인계
```

## 상세 절차

### 1단계: 요청서 작성

인트라넷 > IT지원 > 서버 요청 양식에서 작성합니다.

**필수 기입 항목:**

| 항목 | 설명 | 예시 |
|------|------|------|
| 서비스명 | 사용할 서비스/프로젝트 | 주문 API v2 |
| 용도 | 개발/스테이징/운영 | 운영 |
| 사양 | CPU/메모리/디스크 | 4 vCPU / 16GB / 100GB SSD |
| OS | 운영체제 | Ubuntu 22.04 LTS |
| 네트워크 | 대역/보안그룹 | 내부망, 443/80 오픈 |
| 기한 | 필요 시점 | 2주 내 |
| 비용 부서 | 비용 청구 부서 코드 | DEV-001 |

### 2단계: 승인 프로세스

| 환경 | 승인자 | SLA |
|------|--------|-----|
| 개발 | 팀장 | 1영업일 |
| 스테이징 | 팀장 + 인프라팀장 | 2영업일 |
| 운영 | 본부장 + 인프라팀장 + 보안팀 | 5영업일 |

### 3단계: 인프라팀 구성

- Kubernetes: Namespace + ResourceQuota 생성
- VM: Terraform으로 프로비저닝
- 네트워크: 보안그룹/방화벽 규칙 적용
- 모니터링: [[서비스-모니터링-구성]] 기본 대시보드 연동

### 4단계: 인수인계

- 접속 정보 전달 (안전한 채널)
- 모니터링 대시보드 URL 공유
- 백업 정책 안내

## 자주 묻는 질문

**Q: 급하게 서버가 필요한데 승인이 오래 걸려요.**
A: 개발 환경은 팀장 승인만으로 1영업일 내 가능합니다. 운영은 보안 검토가 필수이므로 최소 5영업일이 소요됩니다.

**Q: 기존 서버 사양 변경도 이 절차를 따르나요?**
A: 네, 동일한 양식에서 "변경 요청"으로 작성하시면 됩니다.
