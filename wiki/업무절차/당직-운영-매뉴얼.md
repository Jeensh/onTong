# 당직(On-call) 운영 매뉴얼

## 당직 로테이션

- **주기**: 1주일 단위
- **교대**: 매주 월요일 09:00
- **인원**: Primary 1명 + Secondary 1명
- **스케줄**: Google Calendar "SRE 당직" 캘린더 참조

## 당직자 책임

### Primary

- PagerDuty 알림 **5분 내 응답**
- 장애 발생 시 [[장애대응-플레이북]] 기반 IC 역할 수행
- 야간(22:00~09:00) 알림 대응
- 일일 점검 수행

### Secondary

- Primary 부재 시 백업
- P1/P2 장애 시 추가 투입
- Primary와 **15분 내 연락 불가** 시 자동 인계

## 일일 점검 체크리스트

매일 09:00 / 18:00 두 번 점검합니다:

### 아침 점검 (09:00)

- [ ] [[서비스-모니터링-구성]] Grafana 대시보드 이상 없음
- [ ] 야간 알림 이력 확인 (PagerDuty)
- [ ] Kubernetes Pod 상태 (`kubectl get pods -A | grep -v Running`)
- [ ] DB Replication lag 정상 (< 1초)
- [ ] 디스크 사용량 90% 미만
- [ ] 백업 정상 완료 확인

### 저녁 점검 (18:00)

- [ ] 에러율 추이 확인
- [ ] 배포 이력 확인 (당일 배포 건 정상 여부)
- [ ] 스케줄링된 배치 작업 완료 확인
- [ ] PagerDuty 알림 채널 정상 동작 확인

## 야간 대응 기준

| 알림 등급 | 야간 대응 | 비고 |
|-----------|-----------|------|
| P1 | 즉시 대응 | IC 역할 수행, 경영진 보고 |
| P2 | 30분 내 확인, 필요시 대응 | 우회 가능하면 다음 날 처리 |
| P3 | 다음 영업일 처리 | 알림만 확인 |
| P4 | 무시 | 야간 알림 대상 아님 |

## 인수인계

교대 시 다음을 전달합니다:

1. **진행 중인 이슈**: 미해결 장애/경고
2. **예정된 작업**: 배포, 마이그레이션, 점검
3. **특이사항**: 일시적 알림 음소거 등

인수인계는 슬랙 `#oncall-handoff` 채널에 기록합니다.
