# 보안정책 가이드

## 적용 범위

본 정책은 전사 IT 시스템을 사용하는 **모든 임직원**에게 적용됩니다.

## 계정 및 인증

### 비밀번호 정책

| 항목 | 기준 |
|------|------|
| 최소 길이 | 12자 이상 |
| 복잡도 | 영문 대/소문자 + 숫자 + 특수문자 |
| 변경 주기 | 90일 |
| 이전 비밀번호 재사용 | 최근 5개 불가 |
| 실패 잠금 | 5회 실패 시 30분 잠금 |

### 2단계 인증 (2FA)

다음 시스템은 **2FA 필수**입니다:

- VPN 접속
- 클라우드 콘솔 (AWS, GCP)
- GitHub/GitLab
- 운영 DB 접속 도구
- 관리자 포털

> 인증 앱: Google Authenticator 또는 회사 지급 하드웨어 키

## 네트워크 접근 제어

### VPN 정책

- 사내망 외부에서는 **반드시 VPN 사용**
- VPN 세션 타임아웃: 8시간
- Split tunneling: **비허용**

### 방화벽 규칙

```
운영 서버:
  - 인바운드: VPN 대역 + 로드밸런서만 허용
  - SSH: Bastion 호스트 경유만 허용 (직접 SSH 금지)

DB 서버:
  - 인바운드: 애플리케이션 서버 대역만 허용
  - 관리 접근: DBA 전용 Bastion 경유
```

## 데이터 보호

### 데이터 등급 분류

| 등급 | 설명 | 예시 | 취급 |
|------|------|------|------|
| **기밀** | 유출 시 법적/사업 피해 | 개인정보, 결제정보 | 암호화 필수, 접근 로그 |
| **내부** | 사내 공유 가능 | 운영 매뉴얼, 시스템 구성 | 외부 반출 금지 |
| **공개** | 외부 공개 가능 | 제품 소개, 공지 | 제한 없음 |

### 암호화 기준

- **전송 중**: TLS 1.2 이상 필수
- **저장 시**: 기밀 데이터는 AES-256 암호화
- **DB**: 개인정보 컬럼 암호화 (애플리케이션 레벨)
