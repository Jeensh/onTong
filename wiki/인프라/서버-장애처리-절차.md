# 서버 장애처리 절차

## 1. 장애 감지
- Grafana 알림 확인
- PagerDuty 호출 여부 확인

## 2. 초기 대응 (5분 이내)
1. 장애 범위 파악 (단일 서버 / 클러스터)
2. 슬랙 #incident 채널에 공유
3. 온콜 담당자에게 에스컬레이션

## 3. 복구
- 서버 재시작: `systemctl restart app`
- 로그 확인: `journalctl -u app --since "10 min ago"`
- 헬스체크 확인 후 트래픽 복원

## 4. 후속 조치
- 포스트모템 작성 (48시간 이내)
- 재발 방지 대책 수립
