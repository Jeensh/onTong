# 서비스 모니터링 구성

## 모니터링 스택

| 구성 요소 | 도구 | 용도 |
|-----------|------|------|
| 메트릭 수집 | Prometheus | CPU, 메모리, 커스텀 메트릭 |
| 시각화 | Grafana | 대시보드, 알림 규칙 |
| 로그 | Loki + Promtail | 중앙 집중 로그 |
| 트레이싱 | Jaeger | 분산 추적 |
| 알림 | AlertManager | 슬랙/PagerDuty 연동 |
| 업타임 | Blackbox Exporter | HTTP/TCP 헬스체크 |

## 핵심 대시보드

### 1. Service Overview (service-overview)

전사 서비스 상태를 한눈에 보는 대시보드:

- **Traffic**: 초당 요청 수 (RPS)
- **Errors**: 5xx 에러율
- **Duration**: P50/P95/P99 응답시간
- **Saturation**: CPU/메모리 사용률

### 2. Database Dashboard (db-dashboard)

- 커넥션 풀 사용률
- 슬로우 쿼리 (>1초)
- Replication lag
- 디스크 사용량 추이

### 3. Kubernetes Dashboard (k8s-dashboard)

- Pod 상태 (Running/Pending/CrashLoopBackOff)
- 노드 리소스 사용률
- HPA 스케일링 이벤트
- PVC 디스크 사용량

## 알림 규칙

| 알림 | 조건 | 등급 | 채널 |
|------|------|------|------|
| High Error Rate | 5xx > 5% (5분간) | P1 | #incident-war-room |
| Slow Response | P95 > 3초 (10분간) | P2 | #alerts-infra |
| High CPU | CPU > 85% (15분간) | P2 | #alerts-infra |
| Disk Full | 디스크 > 90% | P2 | #alerts-infra |
| Pod CrashLoop | 재시작 > 5회/10분 | P3 | #alerts-k8s |
| SSL Expiry | 만료 30일 이내 | P3 | #alerts-infra |
| DB Replication Lag | > 10초 | P2 | #alerts-db |
