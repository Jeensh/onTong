# 모니터링 알림 설정

## 모니터링 도구
- **Grafana**: 대시보드 및 시각화 (grafana.internal)
- **Prometheus**: 메트릭 수집
- **Alertmanager**: 알림 발송

## 알림 규칙

| 항목 | Warning | Critical | 알림 채널 |
|------|---------|----------|-----------|
| CPU 사용률 | > 70% (5분) | > 90% (3분) | Slack + SMS |
| 메모리 사용률 | > 80% | > 95% | Slack + SMS |
| 디스크 사용률 | > 80% | > 90% | Slack |
| API 응답시간 | > 1s (avg) | > 3s (avg) | Slack + PagerDuty |
| DB 커넥션 풀 | > 80% | > 95% | Slack + SMS |

## 에스컬레이션 정책
1. Warning → Slack 알림 → 담당자 확인
2. Critical → SMS + PagerDuty → 15분 내 조치
3. 30분 미복구 → 팀장 에스컬레이션
