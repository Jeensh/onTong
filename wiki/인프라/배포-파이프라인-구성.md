# 배포 파이프라인 구성

## 전체 파이프라인

```
코드 Push → GitHub Actions (CI) → Docker Build → ECR Push
  → ArgoCD Sync (CD) → Staging 배포 → Canary 배포 → Production 전환
```

## CI 단계 (GitHub Actions)

### 트리거 조건

| 브랜치 | 이벤트 | 동작 |
|--------|--------|------|
| `feature/*` | push | lint + test |
| `develop` | push | lint + test + build + staging 배포 |
| `main` | push (merge) | lint + test + build + production 배포 |
| `release/*` | tag | 릴리즈 빌드 + 아티팩트 저장 |

### 주요 Job

1. **lint**: ESLint/Pylint 코드 품질 검사
2. **test**: 단위 테스트 + 통합 테스트
3. **build**: Docker 이미지 빌드 + ECR 푸시
4. **scan**: Trivy 이미지 취약점 스캔

## CD 단계 (ArgoCD)

### GitOps 구조

```
infra-manifests/
├── base/
│   ├── deployment.yaml
│   ├── service.yaml
│   └── kustomization.yaml
├── overlays/
│   ├── staging/
│   └── production/
```

### Canary 배포 전략

1. **5% 트래픽**으로 새 버전 배포 (10분 관찰)
2. **에러율 < 1%** + **P95 < 기존 대비 120%** 확인
3. 통과 시 **50% → 100%** 단계적 전환
4. 실패 시 자동 롤백 (Argo Rollouts)

## 환경 변수 관리

| 유형 | 저장소 | 예시 |
|------|--------|------|
| 일반 설정 | ConfigMap | LOG_LEVEL, FEATURE_FLAGS |
| 시크릿 | Sealed Secrets | DB_PASSWORD, API_KEY |
| 환경별 설정 | Kustomize overlay | REPLICAS, RESOURCE_LIMITS |
