# 캐싱 전략 가이드

## 1. 캐싱 레이어 구성
- L1: 애플리케이션 메모리 캐시 (Caffeine)
- L2: 분산 캐시 (Redis Cluster)
- L3: CDN 캐시 (정적 자원)

## 2. TTL 설정 기준
| 데이터 유형 | 권장 TTL | 근거 |
|------------|----------|------|
| 사용자 세션 | 30분 | 보안 정책 |
| 마스터 데이터 | 1시간 | 변경 빈도 낮음 |
| 검색 결과 | 5분 | 실시간성 필요 |

## 3. 캐시 무효화 전략
- Write-through: 데이터 변경 시 즉시 캐시 갱신
- Event-driven: Kafka 이벤트로 캐시 무효화
