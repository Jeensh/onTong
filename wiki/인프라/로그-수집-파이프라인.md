# 로그 수집 파이프라인

## 아키텍처

```
App Pod → stdout/stderr → Promtail (DaemonSet) → Loki → Grafana
                                                     ↘ S3 (장기 보관)
```

## 로그 레벨 가이드

| 레벨 | 용도 | 예시 |
|------|------|------|
| `ERROR` | 즉시 확인 필요 | DB 연결 실패, 외부 API 타임아웃 |
| `WARN` | 잠재적 문제 | 재시도 발생, 캐시 미스율 높음 |
| `INFO` | 정상 비즈니스 흐름 | 주문 생성, 결제 완료 |
| `DEBUG` | 개발/디버깅 (운영 OFF) | 함수 파라미터, SQL 쿼리 |

> **운영 환경 기본 레벨: `INFO`** (DEBUG는 장애 시에만 일시 활성화)

## 구조화 로그 규칙

모든 서비스는 JSON 형식으로 로그를 출력합니다:

```json
{
  "timestamp": "2026-03-20T14:30:00Z",
  "level": "ERROR",
  "service": "order-api",
  "trace_id": "abc123",
  "message": "Payment gateway timeout",
  "duration_ms": 5023,
  "error_code": "PG_TIMEOUT"
}
```

### 필수 필드

| 필드 | 설명 |
|------|------|
| `timestamp` | ISO 8601 (UTC) |
| `level` | ERROR/WARN/INFO/DEBUG |
| `service` | 서비스 이름 |
| `trace_id` | 분산 추적 ID (Jaeger 연동) |
| `message` | 사람이 읽을 수 있는 메시지 |

## Grafana에서 로그 조회

```logql
# 특정 서비스 에러 로그
{namespace="production", app="order-api"} |= "ERROR"

# trace_id로 추적
{namespace="production"} |= "trace_id=abc123"

# 최근 5분 에러 카운트
count_over_time({namespace="production"} |= "ERROR" [5m])
```

|------|-----------|---------|
