# 캐시 장애 대응 매뉴얼

## 1\. Redis 캐시 장애 유형

### 1.1 메모리 초과 (OOM)

-   `maxmemory` 설정 확인: `redis-cli config get maxmemory`
    
-   eviction policy 확인: `allkeys-lru` 권장
    

### 1.2 연결 풀 고갈

-   최대 연결 수: 기본 10,000
    
-   `CLIENT LIST` 명령으로 현재 연결 확인
    

## 2\. 복구 절차

1.  Redis 프로세스 상태 확인
    
2.  메모리 사용량 점검
    
3.  필요 시 캐시 플러시 (`FLUSHDB`)
    
4.  애플리케이션 연결 풀 재시작
