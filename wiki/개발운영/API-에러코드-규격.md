# API 에러코드 규격

## 에러 응답 표준 형식

```json
{
  "error": {
    "code": "ERR_ORDER_001",
    "message": "주문을 찾을 수 없습니다",
    "detail": "order_id=12345에 해당하는 주문이 존재하지 않습니다",
    "trace_id": "abc-123-def"
  }
}
```

## 에러코드 체계

```
ERR_{도메인}_{번호}
```

| 도메인 | 접두사 | 범위 |
|--------|--------|------|
| 인증/인가 | ERR_AUTH | 001~099 |
| 주문 | ERR_ORDER | 001~099 |
| 결제 | ERR_PAY | 001~099 |
| 상품 | ERR_PROD | 001~099 |
| 시스템 | ERR_SYS | 001~099 |

## 에러코드 목록

### 인증 (ERR_AUTH)

| 코드 | HTTP | 메시지 | 원인 |
|------|------|--------|------|
| ERR_AUTH_001 | 401 | 인증 토큰이 만료되었습니다 | JWT 만료 |
| ERR_AUTH_002 | 403 | 해당 리소스에 접근 권한이 없습니다 | RBAC 거부 |

### 주문 (ERR_ORDER)

| 코드 | HTTP | 메시지 | 원인 |
|------|------|--------|------|
| ERR_ORDER_001 | 404 | 주문을 찾을 수 없습니다 | 잘못된 주문 ID |
| ERR_ORDER_002 | 409 | 이미 취소된 주문입니다 | 중복 취소 시도 |

### 결제 (ERR_PAY)

| 코드 | HTTP | 메시지 | 원인 |
|------|------|--------|------|
| ERR_PAY_001 | 502 | 결제 게이트웨이 응답 시간 초과 | PG사 타임아웃 |

### 시스템 (ERR_SYS)

| 코드 | HTTP | 메시지 | 원인 |
|------|------|--------|------|
| ERR_SYS_001 | 500 | 내부 서버 오류가 발생했습니다 | 예상치 못한 예외 |

## 에러코드 추가 규칙

1. 새 에러코드는 반드시 이 문서에 등록 후 사용
2. HTTP 상태 코드는 RFC 7231 준수
3. `message`는 사용자에게 표시되므로 한국어로 작성
4. `detail`은 개발자 디버깅용 (로그에만 기록, 프로덕션 응답에서 숨김 가능)
