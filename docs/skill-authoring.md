# 스킬 작성 가이드

> Section 1(Wiki) AI Copilot의 **스킬**은 에이전트가 특정 상황에서 따라야 할 워크플로우, 규칙, 출력 형식을 정의하는 마크다운 파일입니다.

스킬은 `wiki/_skills/` 폴더에 마크다운 파일로 정의합니다. **6-Layer 구조**를 따르면 AI 답변 품질이 높아집니다.

---

## 1. 스킬 파일 구조

```markdown
---
type: skill
description: 장애 발생 시 등급 판정부터 초기 대응까지 단계별 안내
trigger:
  - 장애
  - 장애 대응
  - 서비스 장애
  - 서버 다운
  - 에러율
icon: 🚨
scope: shared
enabled: true
category: 장애대응
priority: 9
pinned: true
---

# 장애 초기대응 도우미

## 역할                          ← Layer 1: AI의 페르소나 정의
당신은 숙련된 SRE 엔지니어입니다.
- 톤: 침착하고 명확하게, 행동 지향적으로 안내
- 금지 표현: "아마도", "~일 수 있습니다" (장애 상황에서 모호한 표현 금지)

## 워크플로우                     ← Layer 2: 단계별 처리 절차
### 1단계: 증상 파악
사용자에게 다음을 확인하세요:
- 어떤 서비스가 영향을 받고 있는가?
- 언제부터 발생했는가?

### 2단계: 등급 판정
참조 문서의 기준에 따라 등급을 판정하고 근거를 설명하세요.

### 3단계: 대응 가이드
등급에 맞는 구체적 행동 지침을 제공하세요.

## 지시사항                       ← Layer 3: 구체적 행동 규칙
사용자가 장애 상황을 설명하면 다음 순서로 안내하세요:
1. 현재 증상을 기반으로 장애등급(P1~P4)을 판정하세요
2. 해당 등급의 에스컬레이션 채널과 보고 대상을 안내하세요
3. 즉시 확인해야 할 모니터링 항목을 알려주세요
4. 최근 배포가 있었는지 확인하고, 있다면 롤백 여부를 판단하게 하세요

## 체크리스트                     ← Layer 4: 필수/금지 항목
### 반드시 포함
- 장애등급 판정 근거
- 에스컬레이션 채널 (슬랙/전화)
- 확인할 모니터링 대시보드 URL
### 언급 금지
- 장애 책임 소재
- 비용/매출 구체적 수치

## 출력 형식                      ← Layer 5: 응답 포맷 강제
1. 🔴 등급 판정: P? — (근거 1줄)
2. 📢 에스컬레이션: (채널 + 대상)
3. 🔍 즉시 확인: (모니터링 항목 3~5개)
4. 🛠 조치 가이드: (단계별)
5. 📝 커뮤니케이션 템플릿

## 제한사항                       ← Layer 6: 안전장치
- 참조 문서에 없는 시스템은 "해당 시스템의 런북을 확인하세요"로 안내
- 등급 판단이 모호하면 높은 등급으로 판정 (안전 우선)

## 참조 문서
- [[장애등급-분류기준]]
- [[장애대응-플레이북]]
- [[서비스-모니터링-구성]]
- [[롤백-절차-가이드]]
```

## 2. Frontmatter 필드

| 필드 | 설명 | 예시 |
| ---- | ---- | ---- |
| `type` | 고정값 `skill` | `skill` |
| `description` | 한 줄 설명 (매칭 점수에 반영) | `장애 대응 가이드` |
| `trigger` | 사용자 질문에서 키워드 감지 → 스킬 자동 활성화 | `[장애, 서버 다운]` |
| `icon` | 스킬 아이콘 (이모지) | `🚨` |
| `scope` | `personal` (본인 전용) / `shared` (팀 전체) | `shared` |
| `enabled` | 스킬 사용 가능 여부 | `true` |
| `category` | 폴더 기반 분류 (UI 사이드바 그룹핑) | `장애대응` |
| `priority` | 1~10, 높을수록 동일 키워드에서 우선 선택 | `9` |
| `pinned` | `true`면 동점 시 최우선 | `true` |
| `allowed-tools` | 스킬 내부에서 호출 가능한 내장 스킬 제한 | `[wiki_search, wiki_read]` |

## 3. 6-Layer 구조

| Layer | 역할 | 예시 |
| ----- | ---- | ---- |
| **Layer 1 — 역할** | AI의 페르소나 정의 | "숙련된 SRE 엔지니어입니다" |
| **Layer 2 — 워크플로우** | 단계별 처리 절차 | "1단계: 증상 파악 → 2단계: 등급 판정" |
| **Layer 3 — 지시사항** | 구체적 행동 규칙 | "다음 순서로 안내하세요" |
| **Layer 4 — 체크리스트** | 필수 포함 / 언급 금지 항목 | "등급 판정 근거 포함, 책임 소재 언급 금지" |
| **Layer 5 — 출력 형식** | 응답 포맷 강제 | "🔴 등급 판정 → 📢 에스컬레이션 → ..." |
| **Layer 6 — 제한사항** | 안전장치 | "모호하면 높은 등급으로 판정 (안전 우선)" |

모두 채울수록 답변 품질이 상승하며, **빈 레이어는 자동으로 스킵**됩니다.

## 4. 참조 문서 (`[[WikiLink]]`)

스킬 본문에 `[[문서명]]` 형식으로 참조 문서를 명시하면, 해당 문서의 내용이 AI 응답 시 자동으로 컨텍스트에 포함됩니다.

```markdown
## 참조 문서
- [[장애등급-분류기준]]      ← 이 문서 전체 내용이 LLM 프롬프트에 자동 주입
- [[장애대응-플레이북]]
```

**주의**: 참조 문서가 많을수록 LLM 토큰이 증가합니다. 핵심 문서 3~5개로 제한하세요.

## 5. Per-Skill 도구 제한 (`allowed-tools`)

User-facing 스킬은 frontmatter `allowed-tools` 필드로 사용 가능한 내장 스킬을 제한할 수 있습니다.

```yaml
---
type: skill
allowed-tools:
  - wiki_search
  - wiki_read
  - llm_generate
---
```

설정하지 않으면 intent(질문/작성/편집) 기반 기본값(`INTENT_ALLOWED_SKILLS`)을 사용합니다.

## 6. 스킬 관리 UI

- **생성/편집**: 프론트엔드 사이드바 → `+ 스킬 추가`
- **자동 매칭**: 사용자 질문에서 trigger 키워드 감지 → 관련 스킬 상단 표시
- **우클릭 메뉴**: 복제, 토글, 삭제, 카테고리 이동
- **드래그앤드롭**: 카테고리(폴더) 간 이동
