# Section 3 (Simulation) 사용 가이드

> **대상**: SCM 현업 담당자  
> **목적**: Slab 설계 프로세스에서 발생하는 에러 원인 파악, What-If 시뮬레이션, 파라미터 실시간 시각화  
> **접속 경로**: 상단 탭 → **[Simulation]** 클릭  
> **문서 버전**: 2026-04-16 (Phase 1 + Phase 1.5 완료 — 시나리오 A/B/C 연동 + Custom Agent Hub + Slab 3D Simulator)

---

## 1. Section 3 개요

Section 3은 세 가지 방식의 시뮬레이션 도구를 제공합니다.

| 도구 | 방식 | 활용 상황 |
|------|------|-----------|
| **시나리오 A / B / C** | AI 채팅 에이전트 | 특정 주문의 에러 원인 탐색, 파급효과 분석, 분할 최적화 |
| **Slab 설계 시뮬레이터** | 파라미터 직접 조작 | "폭을 이만큼 바꾸면 어떻게 되지?" 실시간 What-If |
| **Custom Agent** | 나만의 AI 에이전트 생성·실행 | 반복 업무 자동화, 특정 도메인 문제에 특화된 에이전트 구성 |

**도구들은 서로 연결되어 있습니다.** 에이전트 분석 결과를 클릭 한 번으로 시뮬레이터에서 바로 열거나, 온톨로지 그래프의 주문 노드를 클릭해서 시뮬레이터로 진입할 수 있습니다.

## 3. 화면 구성 — 사이드바 네비게이션

화면 왼쪽에 **사이드바**가 고정되어 있습니다. 모든 기능은 사이드바에서 선택합니다.

```
┌──────────────────┬─────────────────────────────────────────┐
│  사이드바 (좌측)   │  메인 콘텐츠 영역 (우측)                  │
│                  │                                         │
│  주요 시나리오     │  선택된 항목에 따라 변경됨:              │
│  ┌ 시나리오 A     │  · 시나리오 A/B/C → 온톨로지그래프+채팅   │
│  ├ 시나리오 B     │  · Slab 설계     → 파라미터 시뮬레이터   │
│  ├ 시나리오 C     │  · Custom Agent  → 에이전트 실행 채팅   │
│  └ Slab 설계     │  · 채팅으로 만들기 → 에이전트 빌더 채팅   │
│                  │  · 양식으로 만들기 → 에이전트 생성 폼    │
│  ── 구분선 ──     │                                         │
│                  │                                         │
│  Custom Agent    │                                         │
│  ┌ 채팅으로 만들기 │                                         │
│  ├ 양식으로 만들기 │                                         │
│  └ [등록된 에이전트 목록]                                    │
│                  │                                         │
└──────────────────┴─────────────────────────────────────────┘
```

**사이드바 섹션 설명**:

| 섹션 | 내용 |
|------|------|
| **주요 시나리오** | 빌트인 시나리오 A/B/C + Slab 설계 시뮬레이터 |
| **Custom Agent** (헤더 클릭) | 등록된 에이전트 목록 + 생성 허브 이동 |
| **채팅으로 만들기** | AI와 대화하며 에이전트 설계 |
| **양식으로 만들기** | 구조화된 폼으로 에이전트 직접 정의 |
| **등록된 에이전트** | 생성된 에이전트 목록 — 클릭하면 바로 실행 |

시나리오 전환 시 이전 대화 내역은 초기화됩니다.

## 4. AI 에이전트 시나리오 (A / B / C)

### 4-1. 레이아웃

```
┌──────────────────────────────────────────────────────────────┐
│  [시나리오 A]  [시나리오 B]  [시나리오 C]  |  [🧊 Slab 설계]  │
├─────────────────────────────────┬────────────────────────────┤
│                                 │                            │
│       Ontology Graph            │       AI 채팅 패널          │
│   (설비·주문 관계 그래프)          │   (질문 입력 → 에이전트)    │
│                                 │                            │
│   · Order 노드 클릭              │   · 추론 과정 / Tool 호출   │
│     → Slab 설계 3D로 이동        │   · 마크다운 응답 스트리밍   │
│   · 탐색 경로 하이라이트           │                            │
│     (에이전트 실행 시 자동)        │   ──────────────────────  │
│                                 │   에이전트 완료 후:          │
│                                 │   [🧊 Slab 설계 3D 연결]   │
│                                 │    버튼 자동 표시            │
└─────────────────────────────────┴────────────────────────────┘
```

---

### 4-2. 시나리오 A — 🔍 폭 범위 산정 불가 (DG320)

**언제 사용**: 특정 주문이 폭 범위 계산 단계에서 `DG320` 에러가 난 경우

**진행 방법**:
1. **[시나리오 A]** 탭 클릭
2. 채팅 입력창에 아래와 같이 입력

   ```
   주문 ORD-2024-0042가 폭 범위 계산에서 DG320 에러났어.
   왜 그러는지 찾고, 폭을 어떻게 조정하면 되는지 알려줘.
   ```

3. AI 에이전트가 자동으로:
   - 주문 데이터 조회 (Tool 호출)
   - 설비 제약(Edging 기준) 확인
   - 에러 원인 분석
   - 파라미터 조정 방안 제시

4. 분석 중 **왼쪽 Ontology Graph**에서 탐색 경로가 하이라이트됩니다
5. 분석 완료 후 채팅 하단에 **Slab 설계 3D 연결 버튼**이 표시됩니다

#### 🧊 Slab 설계 3D 연결

에이전트 분석이 완료되면 채팅 패널 하단에 아래 버튼이 자동으로 표시됩니다.

```
┌──────────────────────────────────────────────────────────────┐
│  🧊 이 주문을 Slab 설계 3D로 확인  (조정 폭 1,570mm)          │
└──────────────────────────────────────────────────────────────┘
```

버튼을 클릭하면:
- **[🧊 Slab 설계]** 탭이 자동으로 열립니다
- 조정된 폭(1,570mm)이 적용된 상태로 3D 뷰어가 세팅됩니다
- 우측 영향도 패널에서 SEQ 전체 ✅ 통과 여부를 즉시 확인할 수 있습니다

---

### 4-3. 시나리오 B — 📊 Edging 기준 변경 파급효과

**언제 사용**: 열연 라인의 Edging 능력 변경 시 기존 주문에 미치는 영향을 미리 분석할 때

**진행 방법**:
1. **[시나리오 B]** 탭 클릭
2. 채팅 입력창에 아래와 같이 입력

   ```
   열연 A라인 Edging 최대 능력을 180mm에서 160mm로 줄이면,
   현재 설계 중인 주문들 중 몇 개나 폭 범위가 깨져?
   ```

3. AI 에이전트가:
   - 현재 Edging 기준 조회
   - 영향받는 주문 목록 추출
   - 각 주문별 폭 범위 재계산
   - 파급효과 요약 제공

#### 🧊 영향받은 슬랩 3D 확인

분석 완료 후 영향받은 슬랩이 있으면 채팅 하단에 버튼이 표시됩니다.

```
┌──────────────────────────────────────────────────────────────┐
│  🧊 영향받은 슬랩 Slab 설계 3D로 확인  (N개 중 첫 번째 · Xmm) │
└──────────────────────────────────────────────────────────────┘
```

버튼을 클릭하면:
- **[🧊 Slab 설계]** 탭이 자동으로 열립니다
- 영향받은 슬랩의 파라미터로 3D 뷰어가 세팅됩니다
- 실제 설계 영향도(SEQ 체크)를 즉시 확인할 수 있습니다

---

### 4-4. 시나리오 C — ⚙️ 단중·매수 최적화

**언제 사용**: 분할수 결정 후 단중 만족률이 낮을 때 더 나은 조합을 찾을 때

**진행 방법**:
1. **[시나리오 C]** 탭 클릭
2. 채팅 입력창에 아래와 같이 입력

   ```
   주문 ORD-2024-0055 Slab 설계 결과 분할수 3으로 나왔는데,
   단중 만족률이 60%밖에 안 돼. 더 나은 조합 없어?
   ```

3. AI 에이전트가 분할수·매수 조합을 최적화하여 단중 만족률을 높이는 방안을 제시합니다

#### 🧊 3D 분할 시각화 딥링크 (신규)

에이전트 분석이 완료되면 채팅 패널 하단에 아래 버튼이 자동으로 표시됩니다.

```
┌──────────────────────────────────────────────────┐
│  🧊 최적 분할수 N개를 Slab 설계 3D로 확인          │
└──────────────────────────────────────────────────┘
```

버튼을 클릭하면:
- **[🧊 Slab 설계]** 탭이 자동으로 열립니다
- 최적 분할수(N개)가 적용된 상태로 3D 뷰어가 세팅됩니다
- 3D 뷰어에서 N개로 분리된 Slab 애니메이션이 즉시 재생됩니다

---

### 4-5. 온톨로지 그래프에서 시뮬레이터 열기 (신규)

왼쪽 Ontology Graph 패널에서 **Order 노드**(점선 링 표시)를 클릭하면 해당 주문의 파라미터로 Slab 설계 3D 시뮬레이터가 자동으로 열립니다.

```
Ontology Graph에서 Order 노드 클릭
      ↓
[🧊 Slab 설계] 탭 자동 이동
      ↓
해당 주문 파라미터(폭/길이/단중/분할수 등) 자동 로딩
      ↓
SEQ 계산 결과 즉시 표시
```

**Order 노드 식별 방법**:
- 노드 호버 시 "클릭하여 시뮬레이터에서 열기"라는 툴팁이 표시됩니다
- ERROR 상태 주문은 빨간색으로 표시되어 즉시 구별됩니다

### 5-2. 파라미터 컨트롤러 (좌측 패널)

#### 주문으로 불러오기 (신규)

좌측 패널 최상단에 **주문 선택 드롭다운**이 표시됩니다.

```
주문으로 불러오기
┌──────────────────────────────────┐
│  — 주문 선택 —                    ▼│
└──────────────────────────────────┘
```

드롭다운을 열면 현재 등록된 주문 목록이 표시됩니다.

```
— 주문 선택 —
ORD-2024-0042 ⚠ DG320 · 1850mm
ORD-2024-0055 · 1040mm
ORD-2024-0061 · 1100mm
ORD-2024-0078 · 1320mm
ORD-2024-0091 · 980mm
```

주문을 선택하면:
- 해당 주문의 목표폭/목표길이/단중/분할수/실수율이 슬라이더에 자동 세팅됩니다
- 두께는 연주설비 mold 기준(250mm)으로 고정됩니다
- 선택한 주문의 설계 상태(✅ 설계완료 / ❌ 에러)가 드롭다운 아래에 표시됩니다

> **활용 팁**: ORD-2024-0042(DG320 에러 주문)를 선택하면 목표폭 1,850mm로 세팅되어 즉시 에러 상태를 확인할 수 있습니다.

---

#### 파라미터 슬라이더

6개 파라미터를 **슬라이더** 또는 **숫자 직접 입력**으로 조작합니다.

| 파라미터 | 단위 | 기본값 | 설비 범위 | 설명 |
|---------|------|--------|---------|------|
| 목표폭 | mm | 1,040 | 900 ~ 1,570 | 1차 폭범위 산정 기준 |
| 두께 | mm | 250 | 200 ~ 300 | 연주설비 mold 두께 |
| 목표길이 | mm | 11,700 | 4,500 ~ 12,000 | 1차 길이범위 산정 기준 |
| 단중 | kg | 23,800 | 8,000 ~ 30,000 | 분할수 결정 핵심 제약 |
| 분할수 | 개 | 2 | 1 ~ 5 | Slab → 코일 분할 수 |
| 실수율 | — | 0.943 | 0.85 ~ 0.99 | 매수 산정 공식 사용 |

#### 파라미터 상태 인디케이터

각 파라미터 이름 왼쪽 점으로 현재 상태를 즉시 확인합니다.

| 색상 | 의미 |
|------|------|
| 🟢 녹색 | 정상 범위 내 |
| 🟡 노란색 | 경계 근접 (범위 끝 5% 이내) |
| 🔴 빨간색 | 범위 초과 — 설계 불가 상태 |

#### 입력 방법

**슬라이더 드래그**: 슬라이더를 좌우로 드래그하면 값이 즉시 변경됩니다.

**숫자 직접 입력**: 슬라이더 오른쪽 입력란을 클릭 후 숫자 입력 → Enter 또는 클릭 해제 시 즉시 반영.

### 5-3. 3D Slab Viewer (중앙 패널)

#### 기본 조작

| 동작 | 방법 |
|------|------|
| 회전 | 마우스 왼쪽 버튼 드래그 |
| 줌 | 마우스 휠 스크롤 |

#### 색상 코딩

| Slab 색상 | 상태 | 의미 |
|-----------|------|------|
| 🔵 스틸 블루 | 정상 | 모든 SEQ 통과, 설계 가능 |
| 🟠 주황 | 경고 | 1개 이상 파라미터 경계 근접 |
| 🔴 빨강 + 진동 | 에러 | 설비 범위 초과, 설계 불가 |
| 🟣 연보라 + 펄스 | 계산 중 | API 응답 대기 중 |

#### 분할수 시각화

분할수 슬라이더를 1 → 2 → 3으로 올리면 Slab이 Y축 방향으로 분리되는 모습이 애니메이션으로 표시됩니다. 각 조각 위에 **분할 단중(kg)** 라벨이 표시됩니다.

#### 치수 라벨

3D Slab 주변에 현재 치수가 표시됩니다.
- X축 방향 → 폭 (mm)
- Y축 방향 → 두께 (mm)
- Z축 방향 → 길이 (mm)

---

### 5-4. 영향도 분석 패널 (우측 패널)

파라미터 변경과 동시에 실시간으로 **설계 프로세스 9단계**를 계산하고 결과를 표시합니다.

#### 상단: 설계 가부 배지

```
✅ 설계 가능          통과 SEQ: 9/9
```

오류 시:
```
🔴 설계 불가          통과 SEQ: 6/9
```

#### 중간: 계산 요약

| 항목 | 표시 예시 |
|------|---------|
| 1차 폭범위 | 960 ~ 1,110 mm |
| 1차 길이범위 | 4,500 ~ 12,000 mm |
| 1차 단중범위 | 8,445 ~ 26,040 kg |
| 분할 단중 | 11,900 kg/분할 |
| 산정 매수 | 3 매 |

#### 하단: SEQ별 체크 트리

```
✅ SEQ2.  두께 결정            250mm ← 연주설비 범위 내
✅ SEQ3.  1차 폭범위 산정      960 ~ 1,110mm
           └── 목표폭 1,040mm → 범위 내
✅ SEQ4.  1차 길이범위 산정    4,500 ~ 12,000mm
✅ SEQ5.  1차 단중범위 산정    8,445 ~ 26,040kg
✅ SEQ8.  분할수 결정          분할수 2 ≤ 최대 분할수 4
✅ SEQ9.  매수 산정            3매
✅ SEQ12. 2차 폭범위 산정      1,022 ~ 1,057mm
✅ SEQ14. Target 폭 확정       1,040mm
✅ SEQ16. Target 길이 확정     11,700mm
```

각 SEQ 행을 **클릭**하면 계산 상세 내역이 펼쳐집니다.

#### 에러 강조 표시

범위 초과 시 패널 하단에 연쇄 실패 내역이 강조됩니다.

```
🔴 연쇄 실패 감지
  SEQ3  1차 폭범위 산정 → 목표폭 1,850mm — HR-A 최대폭 1,570mm 초과
  SEQ12 2차 폭범위 산정 → 목표폭 1,850mm — 2차 폭범위 이탈
  SEQ14 Target 폭 확정  → 1,850mm 확정 불가 → 1,570mm 권장
```

### 5-5. 변경 전/후 비교 (중앙 하단)

현재 파라미터를 기준점으로 저장한 뒤 조작하면 **변화량 비교 테이블**이 표시됩니다.

**사용 방법**:

1. 기본값 상태에서 좌측 패널 하단 **[이 설정 저장]** 버튼 클릭
2. 파라미터를 변경
3. 중앙 패널 하단에 비교 테이블 자동 표시

**표시 예시**:

| 항목 | 변경 전 | 현재 | 변화 |
|------|--------|------|------|
| 목표폭 | 1,040 mm | 1,300 mm | ↑ 증가 |
| 목표길이 | 11,700 mm | 10,000 mm | ↓ 감소 |
| 1차 폭범위 | 960~1,110 | 1,120~1,390 | 변경됨 |
| 설계 가부 | ✅ 가능 | ✅ 가능 | 변화 없음 |
| 산정 매수 | 3 매 | 3 매 | 변화 없음 |

| **이 설정 저장** | 현재 파라미터를 스냅샷으로 저장 → 비교 테이블 활성화 |

## 6. 연동 흐름 — 에이전트 ↔ 시뮬레이터

Phase 1에서 구현된 핵심 연동 흐름 4가지입니다.

### 흐름 1. Scenario A 에러 분석 → Slab 설계 3D 검증

```
1. [시나리오 A] 탭 → "ORD-2024-0042 DG320 에러" 입력
2. 에이전트 분석 완료 (좌측 그래프에서 탐색 경로 하이라이트)
3. [🧊 이 주문을 Slab 설계 3D로 확인 (조정 폭 1,570mm)] 버튼 클릭
4. [🧊 Slab 설계] 탭 자동 이동 + 목표폭 1,570mm 세팅
5. SEQ 전체 ✅ → 조정된 파라미터로 설계 가능함을 즉시 확인
```

### 흐름 2. Scenario B 파급효과 분석 → 영향받은 슬랩 3D 확인

```
1. [시나리오 B] 탭 → "Edging 180→160mm 변경 파급효과" 입력
2. 에이전트: 영향받은 주문 목록 제시
3. [🧊 영향받은 슬랩 Slab 설계 3D로 확인 (N개 중 첫 번째)] 버튼 클릭
4. [🧊 Slab 설계] 탭 자동 이동 + 영향받은 슬랩 파라미터 세팅
5. 설계 영향도 (SEQ 체크) 즉시 확인
```

### 흐름 3. Scenario C 최적화 → 3D 분할 확인

```
1. [시나리오 C] 탭 → "ORD-2024-0055 분할수 최적화" 입력
2. 에이전트: "최적 분할수 N개" 결과 제시
3. [🧊 최적 분할수 N개를 Slab 설계 3D로 확인] 버튼 클릭
4. [🧊 Slab 설계] 탭 자동 이동 + 분할수 N 세팅
5. 3D 뷰어에서 N개 분리 애니메이션 즉시 재생
```

### 흐름 4. 온톨로지 그래프 탐색 → 주문 시뮬레이션

```
1. [시나리오 A/B/C] 탭에서 에이전트 실행
2. 좌측 Ontology Graph에서 Order 노드 확인 (점선 링 표시)
3. 관심 있는 Order 노드 클릭 (예: ORD-2024-0042)
4. [🧊 Slab 설계] 탭 자동 이동 + 해당 주문 파라미터 세팅
5. 시뮬레이터에서 설계 상태 직접 확인
```

### 흐름 5. 시뮬레이터에서 직접 주문 선택

```
1. [🧊 Slab 설계] 탭 클릭
2. 좌측 상단 [주문 선택] 드롭다운 클릭
3. 원하는 주문 ID 선택 (ORD-2024-XXXX)
4. 해당 주문 파라미터 자동 로딩
5. 상태 확인: ❌ DG320 에러면 목표폭 슬라이더가 빨간색으로 표시됨
```

## 7. Custom Agent — 나만의 Slab 설계 에이전트

Custom Agent는 Slab 설계 도메인에 특화된 나만의 AI 에이전트를 만들고 등록하여 반복적으로 사용하는 기능입니다.

### 7-1. 언제 사용하나요?

| 상황 | 활용 예시 |
|------|---------|
| **반복 업무 자동화** | "DG320 에러 주문이 생기면 항상 같은 진단 절차를 따른다" → 에이전트 등록 |
| **특정 담당자 특화** | 폭 범위 담당자 / 단중 최적화 담당자 / 파급효과 분석 담당자별 에이전트 |
| **신속 조회** | "특정 주문 ID만 입력하면 즉시 설비 기준까지 확인해주는 에이전트" |

---

### 7-2. Custom Agent Hub 접속

사이드바에서 **"Custom Agent"** 섹션 헤더를 클릭합니다.

```
사이드바
  ── Custom Agent ──  ← 이 헤더를 클릭
  ┌ 채팅으로 만들기
  └ 양식으로 만들기
```

헤더 클릭 시 우측 메인 영역에 **에이전트 허브** 화면이 열립니다.

### 7-3. 에이전트 생성 — 채팅으로 만들기

가장 쉬운 방법입니다. AI와 자유롭게 대화하며 에이전트를 설계합니다.

**접속 방법**: 사이드바 **"채팅으로 만들기"** 클릭

**진행 흐름**:

```
1. AI: "어떤 Slab 설계 문제를 해결하고 싶으신가요?"
      ↓
2. 사용자: "DG320 에러가 난 주문을 자동으로 진단하고
            가능한 폭을 제안해주는 에이전트를 만들고 싶어"
      ↓
3. AI: 몇 가지 확인 질문 (필요 도구, 이름, 예시 입력 등)
      ↓
4. AI: 에이전트 정의 완성 → 미리보기 카드 표시
      ↓
5. 사용자: [🤖 이 Agent 등록하기] 버튼 클릭
      ↓
6. 에이전트 등록 완료 → 즉시 실행 화면으로 이동
```

**대화 예시**:

```
나: DG320 에러가 난 주문의 폭 범위 에러 원인을 자동으로
    찾아주는 에이전트를 만들고 싶어.

AI: 알겠습니다! 폭 범위 진단 에이전트를 만들어 드리겠습니다.
    몇 가지 질문을 드려도 될까요?
    
    ① 에이전트 이름은 어떻게 할까요? (예: "DG320 자동 진단")
    
나: "DG320 진단기"로 해줘.

AI: 좋습니다. 어떤 도구들이 필요할까요?
    - 주문 정보 조회 (get_order_info)
    - 폭 범위 시뮬레이션 (simulate_width_range)
    - 폭 조정 제안 (suggest_adjusted_width)
    
나: 세 개 다 필요해.

AI: 완성되었습니다! 아래 에이전트를 등록하시겠어요?
    [미리보기 카드 표시]
```

**등록 후**: 사이드바에 새 에이전트가 추가되고, 클릭하면 바로 실행할 수 있습니다.

### 7-4. 에이전트 생성 — 양식으로 만들기

항목을 직접 입력하여 에이전트를 정밀하게 설정합니다.

**접속 방법**: 사이드바 **"양식으로 만들기"** 클릭

**입력 항목**:

| 항목 | 설명 | 예시 |
|------|------|------|
| **아이콘** | 에이전트를 나타낼 이모지 (10개 프리셋) | 🔍 |
| **색상** | 사이드바 및 채팅 강조 색상 (8개 프리셋) | 빨간색 |
| **이름** \* | 에이전트 이름 | DG320 자동 진단 에이전트 |
| **설명** \* | 한 문장 설명 | DG320 에러 주문을 자동으로 진단합니다 |
| **사용 도구** \* | 체크박스로 도구 선택 (7개 중 복수 선택) | 주문 조회, 폭 시뮬레이션, 폭 조정 제안 |
| **시스템 프롬프트** \* | 에이전트 역할과 행동 방식 상세 정의 | 아래 참고 |
| **예시 질문** | 사용자에게 보여줄 예시 (선택) | ORD-2024-0042의 DG320 에러를 분석해줘 |

(\* 필수 입력)

**사용 가능한 도구 7가지**:

| 도구 | 기능 |
|------|------|
| 📋 주문 정보 조회 | 주문 ID로 설계 파라미터 조회 |
| 📐 폭 범위 시뮬레이션 | 목표폭 + 열연라인으로 폭 범위 산정 |
| 🔧 폭 조정 제안 | DG320 에러 주문에 가능한 폭 자동 제안 |
| 🏭 설비 기준 조회 | 연주/열연 설비 기준 (폭/길이/Edging) 조회 |
| 📊 Edging 파급효과 | Edging 능력 변경 시 주문 영향 분석 |
| ⚙️ 분할수 최적화 | 단중 만족률 최대화 분할수 탐색 |
| 🧮 설계 전체 계산 | SEQ 1~16 전체 설계 프로세스 계산 |

**시스템 프롬프트 작성 팁**:

```
당신은 POSCO Slab 설계 전문가입니다.
DG320 에러가 발생한 주문을 진단하고, 자동으로 폭을 조정하여
해결 방안을 제시합니다.

행동 원칙:
- 항상 주문 정보를 먼저 조회하세요
- 설비 기준과 비교하여 에러 원인을 명확히 설명하세요
- 조정 폭은 구체적인 수치(mm 단위)로 제시하세요
- 한국어로 친절하게 답변하세요
```

## 8. 자주 묻는 질문 (FAQ)

### 빌트인 시나리오 / Slab 설계 시뮬레이터

**Q. 에이전트 완료 후 [🧊 시뮬레이터] 버튼이 안 보입니다**  
A. 시나리오 A의 경우 에이전트가 `suggested_width` 값을 추출했을 때만 버튼이 표시됩니다. "ORD-2024-0042 DG320 에러 분석"처럼 구체적인 주문 ID와 에러 코드를 언급해야 정상적으로 표시됩니다.

**Q. 드롭다운에서 주문을 선택해도 파라미터가 안 바뀝니다**  
A. 백엔드 서버(`localhost:8001`)가 실행 중인지 확인하세요. 주문 목록은 서버에서 실시간으로 불러옵니다.

**Q. 좌측 온톨로지 그래프에서 Order 노드를 클릭했는데 Slab 설계 탭이 안 열립니다**  
A. 에이전트를 먼저 실행해야 그래프가 초기화됩니다. 에이전트 실행 후 그래프에 주문 노드가 표시되면 클릭이 가능합니다.

**Q. 3D 뷰어가 안 보입니다**  
A. 페이지 최초 진입 시 Three.js 라이브러리 로딩에 2~3초가 걸릴 수 있습니다. 잠시 기다리면 자동으로 표시됩니다.

**Q. 슬라이더를 움직여도 우측 패널이 업데이트되지 않습니다**  
A. 백엔드 서버(`localhost:8001`)가 실행 중인지 확인하세요.

**Q. Mock 데이터와 실제 설비 수치가 다릅니다**  
A. 현재 Section 3의 모든 수치는 Mock 데이터 기반입니다. Section 2(Modeling) 온톨로지 완성 후 실제 데이터로 대체될 예정입니다.

### Custom Agent

**Q. 채팅으로 에이전트를 만들었는데 [등록하기] 버튼이 안 나옵니다**  
A. AI 에이전트 빌더가 충분한 정보를 수집해야 에이전트 정의가 생성됩니다. 이름, 필요한 도구, 에이전트가 해결할 문제를 명확하게 설명하면 보통 3~5번 대화 후 버튼이 나타납니다. 나타나지 않으면 "이 정도면 에이전트 만들어줘"라고 직접 요청해보세요.

**Q. 에이전트를 등록했는데 사이드바에 목록이 안 보입니다**  
A. 페이지를 새로고침하면 서버에서 목록을 다시 불러옵니다. 앱 시작 시 자동 로드되지만 등록 직후에는 즉시 사이드바에 반영됩니다. 그래도 안 보이면 백엔드 서버 상태를 확인하세요.

**Q. Custom Agent가 빌트인 시나리오 A/B/C와 어떻게 다른가요?**  
A. 빌트인 시나리오는 POSCO 팀이 사전에 설계한 특정 업무 흐름입니다. Custom Agent는 사용자가 직접 역할과 도구를 정의하여 자신의 업무에 최적화된 에이전트를 만드는 기능입니다. 같은 Slab 설계 도구를 사용하지만 어떤 문제에 집중할지를 직접 결정합니다.

**Q. 에이전트를 삭제하면 대화 내역도 사라지나요?**  
A. 네, 에이전트와 함께 해당 에이전트의 대화 내역도 모두 삭제됩니다. 삭제 전 중요한 내용은 별도로 저장해두세요.

**Q. 시스템 프롬프트를 어떻게 작성하면 좋나요?**  
A. 에이전트의 역할, 사용할 도구 순서, 답변 형식을 명시하면 좋습니다. 예: "항상 주문 정보 조회 → 설비 기준 확인 → 에러 원인 분석 순서로 진행하세요." 너무 길면 복잡해질 수 있으니 핵심만 3~5문장으로 작성하세요.

## 9. 주요 에러 코드 및 대처 방법

| 에러 내용 | 원인 | 조정 방법 |
|-----------|------|-----------|
| `HR-A 최대폭 1,570mm 초과` | 목표폭이 열연설비 한계 초과 | 목표폭을 1,570mm 이하로 낮춤 |
| `최소 길이 미달` | 목표길이가 분할 후 최소 코일 길이 미달 | 목표길이 높이거나 분할수 줄임 |
| `단중 최대 초과` | 치수 대비 단중이 너무 큼 | 폭 또는 길이를 줄이거나 단중 수정 |
| `최대 분할수 N 초과` | 현재 단중 대비 분할수가 너무 많음 | 분할수를 N 이하로 줄임 |
| `mold 두께 250mm와 불일치` | 연주설비 mold 두께 고정값 위반 | 두께를 250mm로 복원 |
