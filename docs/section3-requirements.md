## 1. 비즈니스 시나리오 정의

### 1-1. 시나리오 A: 폭 범위 산정 불가 (DG320류 에러) -- 원인 탐색 & 자동 파라미터 조정

> Edging 기준 테이블에 매칭되는 주문이 없어 1차 폭 하한/상한 계산이 불가능해지는 상황에서, AI가 원인 파라미터를 역추적하고, 폭 조정 시뮬레이션을 통해 정상 범위를 찾아준다.

**사용자 입력 예시**
> "주문 ORD-2024-0042가 폭 범위 계산에서 에러났어. 왜 그러는지 찾고, 폭을 어떻게 조정하면 되는지 알려줘."

**에이전트 동작 흐름**
1. 온톨로지 그래프에서 해당 주문 노드 탐색
2. 연결된 Edging 기준 노드 조회 -> 매칭 테이블 확인
3. 불일치 파라미터(목표폭, Edging능력범위) 식별
4. `simulate_width_range(order_width, edging_table)` Tool 반복 호출
5. 정상 범위가 나오는 파라미터 조합 리턴
6. 채팅 패널 하단에 Slab 설계 3D 확인 버튼 표시 -> 클릭 시 Slab 설계 탭으로 이동

---

### 1-2. 시나리오 B: Edging 기준 변경 시 파급 효과 분석 (What-If 시뮬레이션)

> 열연 Edging 능력 기준값이 변경되었을 때, 연결된 Slab 폭 상한/하한에 어떤 영향을 주는지 자동으로 파급 효과(Ripple Effect)를 추적한다.

**사용자 입력 예시**
> "열연 A라인 Edging 최대 능력을 180mm에서 160mm로 줄이면, 현재 설계 중인 주문들 중 몇 개나 폭 범위가 깨져?"

**에이전트 동작 흐름**
1. 변경 파라미터 파싱 (기준: 180 -> 160)
2. 온톨로지 그래프에서 해당 Edging 노드에 연결된 모든 주문 노드 탐색
3. 각 주문별로 `simulate_width_impact(order, new_edging_limit)` Tool 호출
4. 영향받는 주문 목록, 폭 범위 변화량, 설계 가부 집계
5. Ripple Effect 보고서(JSON) 생성
6. 채팅 패널 하단에 영향받은 슬랩 Slab 설계 3D 확인 버튼 표시

---

### 1-3. 시나리오 C: 설계 결과 이상 감지 -- 단중/매수 조합 최적화

> 분할수 결정 Loop에서 설계대기량이 만족되지 않아 최종 단중/매수 조합이 비효율적으로 나올 때, AI가 최적 분할수를 찾아 단중 만족률을 최대화한다.

**사용자 입력 예시**
> "주문 ORD-2024-0055 Slab 설계 결과 분할수 3으로 나왔는데, 단중 만족률이 60%밖에 안 돼. 더 나은 조합 없어?"

**에이전트 동작 흐름**
1. 주문 노드에서 현재 설계결과(단중범위, 매수, 분할수) 조회
2. `simulate_split_combinations(order_weight_range, yield_rate, splits)` Tool 호출 (분할수 1~5 반복)
3. 단중 만족률 MAX인 분할수 조합 선택
4. 설계대기량 만족 여부 최종 검증
5. 최적 결과 JSON 리턴
6. 3D 화면에서 Slab 분할 시뮬레이션 애니메이션 표시 (1개 Slab -> N개 분할)

## 3. 핵심 계산 공식 (SEQ별 설계 로직)

### 3-1. SEQ 2: 두께 결정

- 연주설비 mold 두께 고정 (CC-01: 250mm)

### 3-2. SEQ 3: 1차 폭범위 산정

```
1차 폭범위 lower = target_width - edging_max * 0.444
1차 폭범위 upper = target_width + edging_max * 0.389
```

- 기본값 기준: 960 ~ 1,110mm
- 계수 0.444, 0.389는 임의 근사값 (도메인 검증 필요)

### 3-3. SEQ 4: 1차 길이범위 산정

```
길이범위 lower = max(CC 최소길이, HR 최소길이, 최소코일길이 * 분할수)
길이범위 upper = min(CC 최대길이, HR 최대길이)
```

- 분할수 2일 때: 최소코일길이(2,750) * 2 = 5,500mm

### 3-4. SEQ 5: 1차 단중범위 산정

```
단중 = 비중(7.82) * 폭(mm) * 두께(mm) * 길이(mm) / 1,000,000 (kg)
```

- 기본값 기준 단중범위: 8,445 ~ 26,040 kg
- 단중범위(lower, upper)는 폭범위 x 두께 x 길이범위의 조합으로 산출

### 3-5. SEQ 8: 분할수 결정

```
최대분할수 = floor(단중 / (비중 * 목표폭 * 두께 * 최소코일길이 / 1,000,000))
```

### 3-6. SEQ 9: 매수 산정

```
매수 = ceil(단중 / 8,000)   (단순 Mock -- 실제와 많이 다를 가능성 높음)
```

- 실제 공식 추정: 주문 요구 중량 / (단중 * 실수율)

### 3-7. SEQ 12: 2차 폭범위

```
2차 폭범위 lower = target_width - edging_max * 0.1
2차 폭범위 upper = target_width + edging_max * 0.067
```

- 기본값 기준: 1,022 ~ 1,057mm (1차 폭범위보다 훨씬 좁음)

### 3-8. SEQ 14: Target 폭 확정

- 목표폭이 2차 범위 내에 있으면 그대로 확정

### 3-9. SEQ 16: Target 길이 확정

- 목표길이가 길이범위 내에 있으면 그대로 확정

## 6. 시각화 요구사항

### 6-1. 3D Slab 뷰어

- **라이브러리**: `@react-three/fiber` + `@react-three/drei`
- **기본 형태**: `BoxGeometry(width, thickness, length)` -- X축=폭, Y축=두께, Z축=길이
- **비율**: 실제 치수 비례로 표현 (화면에 맞게 스케일링)
- **카메라**: `OrbitControls`로 마우스 드래그 자유 회전
- **치수 라벨**: `@react-three/drei`의 `<Html>` 컴포넌트로 폭/길이/단중 수치 표시

#### 상태별 색상 코딩

  - 길이 증가 -> Z축으로 앞뒤 대칭 확장
  - 두께 증가 -> Y축으로 위아래 대칭 확장

| 납기 임박 주문 우선 감지 | 오늘 설계해야 할 주문 중 에러 위험도 높은 것 선제 알림 |
| 설비 가동 중단 시 영향 주문 파악 | "HR-A 라인 내일 4시간 중단되면 몇 개 주문에 영향?" |
| 강종 변경 시 파라미터 재계산 | 강종 바뀌면 비중/Edging 기준 자동 재적용 |
| 설계 불가 주문 자동 대안 제시 | 에러난 주문에 대해 "이 설비로 바꾸면 가능합니다" |
| 복수 주문 동시 최적화 | 같은 Cast 내 여러 주문의 Slab 배치를 한 번에 최적화 |

### 9-2. IT 운영자 관점

| 후보 시나리오 | 한 줄 설명 |
|-------------|---------|
| 설계 Job 비정상 종료 원인 탐색 | 어제 밤 설계 배치가 왜 멈췄는지 로그 없이도 원인 추적 |
| 기준 데이터 변경 이력 감지 | Edging 기준 테이블이 언제 바뀌었고 누가 바꿨는지 추적 |
| 설비 기준 변경 예정 시 선제 스캔 | 기준 변경 공지 -> 영향받는 주문 자동 목록화 |

> **검증 필요**: 위 후보 중 실제로 현업에서 쓸 것 같은 시나리오? "이거 되면 진짜 쓴다"고 할 시나리오가 따로 있는가?

## 13. 검증 시나리오 (개발 완료 후 확인)

### 시나리오 1 -- 정상 -> 에러 -> 복구

1. 기본값(폭 1040mm)으로 시작 -> Slab 파란색, 모든 SEQ 녹색
2. 폭 슬라이더를 1850mm로 올림 -> Slab 빨간색 + 진동, SEQ3/12/14 빨간 표시, 경고 메시지 출력
3. 폭을 1200mm로 낮춤 -> Slab 다시 파란색으로 전환, SEQ 전체 녹색 복구

### 시나리오 2 -- 분할수 변화

1. 분할수 1 -> 2 -> 3 조작
2. Slab이 Z축으로 1개 -> 2개 -> 3개로 분리되는 애니메이션 확인
3. 각 조각의 단중 라벨이 분할수에 따라 업데이트 확인

### 시나리오 3 -- 변경 전/후 비교

1. 기본값 상태에서 [이 설정 저장] 클릭
2. 폭 1040 -> 1300, 길이 11700 -> 10000으로 변경
3. 비교 테이블에 변경 전/후 값과 영향 범위 차이가 표시되는지 확인

### 시나리오 4 -- 에이전트 -> 시뮬레이터 전체 흐름

1. 시나리오 A -> "ORD-2024-0042 DG320 에러 분석" 입력 -> 에이전트 완료 -> 딥링크 버튼 표시
2. 버튼 클릭 -> Slab 설계 탭 이동 + 목표폭 1,570mm 자동 세팅 + SEQ 전체 통과
3. 온톨로지 그래프 Order 노드 클릭 -> 시뮬레이터 이동 + 파라미터 로딩

### 시나리오 5 -- 계산 정확도 검증

- 기본값 SEQ별 결과를 실제 설계 담당자에게 보여주고 수치 확인
- 에러 케이스 1: 폭 1,850mm -> SEQ3 에러 + DG320 메시지
- 에러 케이스 2: 길이 4,000mm -> SEQ4 최소 길이 미달 에러
- 에러 케이스 3: 단중 35,000kg -> SEQ5 단중 초과 에러
