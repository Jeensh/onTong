## 1. 프로젝트 개요

**onTong**은 사내 위키 문서를 통합 관리하고, AI 에이전트를 통해 문서 검색·질의응답을 제공하는 **지식 관리 플랫폼**입니다.

### 핵심 가치
- **단일 진실의 원천(SSOT)**: 흩어진 사내 문서를 한 곳에서 관리
- **AI 기반 지식 접근**: 자연어로 질문하면 관련 문서를 찾아 답변
- **문서 신뢰도 관리**: 충돌·중복·폐기 문서를 자동 감지하고 계보 추적
- **문서 간 연결 탐색**: 위키링크·계보·유사도 기반 관계 그래프

### 기술 스택
| 계층 | 기술 |
|------|------|
| Frontend | Next.js 15 (Turbopack), React, Tailwind, Tiptap, Zustand |
| Backend | FastAPI (Python), PydanticAI |
| 벡터DB | ChromaDB (임베딩 저장 + 유사도 검색) |
| LLM | OpenAI GPT-4o-mini (LiteLLM 추상화) |
| 검색 | BM25 + 벡터 하이브리드 (RRF), MiniSearch (클라이언트) |

## 3. 시스템 아키텍처 (데이터 흐름)

```
사용자 입력
    │
    ├─ 문서 편집 ──→ Tiptap 에디터 ──→ PUT /api/wiki/file
    │                                      │
    │                                      ├─→ frontmatter 파싱 + 메타데이터 주입
    │                                      ├─→ ChromaDB upsert (임베딩 + 메타데이터)
    │                                      ├─→ BM25 인덱스 동기화
    │                                      └─→ 쿼리 캐시 무효화
    │
    ├─ AI 질문 ───→ On-Tong Agent ──→ RAG 파이프라인
    │                                      │
    │                                      ├─→ 쿼리 보강 (대화 맥락)
    │                                      ├─→ 하이브리드 검색 (BM25 + 벡터 RRF)
    │                                      ├─→ 메타데이터 필터링 (domain/process)
    │                                      ├─→ Cross-encoder 리랭킹
    │                                      ├─→ Cognitive Reflect (의도→초안→검토)
    │                                      └─→ 최종 답변 생성 (출처 + 충돌 경고)
    │
    ├─ 문서 검색 ──→ Ctrl+K 커맨드 팔레트
    │                  ├─ MiniSearch (클라이언트, 즉시)
    │                  └─ 하이브리드 검색 (서버, 의미)
    │
    └─ 관계 탐색 ──→ 문서 그래프 / 연결 문서 패널 / WikiLink
                       ├─ wiki-link 백링크 맵
                       ├─ lineage (supersedes 체인)
                       ├─ related 메타데이터
                       └─ 임베딩 유사도 (opt-in)
```

## 4. 현재 상태 평가

### 강점
- **문서 작성~검색~AI 질답~관계 탐색**까지 end-to-end 기능 완성
- **문서 충돌/중복 자동 감지** → 관리자 부담 경감
- **증분 인덱싱 + 캐싱** → 실시간 편집-검색 동기화
- **인증 추상화** → SSO/LDAP 즉시 연결 가능한 구조

### 알려진 한계
| 항목 | 현황 | 영향도 |
|------|------|--------|
| 인증 | NoOp (개발용) | 실서비스 전 SSO 연동 필요 |
| 동시 편집 | 미지원 (마지막 저장 우선) | 다수 사용자 시 충돌 가능 |
| 권한 관리 | 없음 (전원 읽기/쓰기) | 민감 문서 접근 제어 불가 |
| 모바일 | 미최적화 | 데스크톱 전용 |
| 배포 | 로컬 개발 환경만 | 프로덕션 인프라 미구성 |
| 대량 데이터 | 15개 샘플 문서 테스트 | 수천 문서 스케일 미검증 |

## 6. 데모 시나리오 (10분)

> 회의 시 라이브 데모용 추천 순서

1. **문서 탐색** — 사이드바에서 문서 열기, 탭 전환
2. **문서 편집** — 마크다운 편집, 표 붙여넣기, 이미지 첨부
3. **WikiLink** — `[[문서명]]` 타이핑 → 클릭해서 이동
4. **검색** — Ctrl+K → 키워드 검색 → 의미 검색 전환
5. **AI 질의** — "출장비 규정 알려줘" → 출처 참조 + 탐색 과정 시각화
6. **문서 그래프** — 관리 → 문서 관계 그래프 → 노드 탐색
7. **충돌 감지** — 문서 충돌 대시보드 → 나란히 비교 → 폐기 처리
8. **태그 관리** — 태그 브라우저 → 미태깅 대시보드 → 자동 태깅
